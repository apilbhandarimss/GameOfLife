What Are We Looking At?

This animation is a cellular automaton known as Conway's Game of Life. It's a zero-player game where the evolution of the system is determined entirely by its initial state. The simulation demonstrates emergent complexity from simple rules.

  • Grid: A 200×300 cell grid where each cell is either alive (green) or dead (black)
  • Rules: Simple rules determine which cells live, die, or are born each generation
  • Patterns: Complex patterns emerge from random initial conditions
  • Graphics: Rendered in C++ using the Raylib graphics library

Understanding the Code

1. Header Files and Macros

The program starts with necessary includes and grid configuration:

Includes and Macros

#include"raylib.h"

#define rows 200
#define col 300
#define cell_size 5

  • iostream - Standard C++ input/output
  • cstdlib - Provides the rand() function for randomization
  • raylib.h - Graphics library for window and drawing operations
  • rows, col - Grid dimensions (200 × 300 cells)
  • cell_size - Each cell is 5×5 pixels on screen

2. Counting Neighbors

This function counts how many living neighbors surround a cell:

countNeig Function 
int countNeig(int r, int c)
{
    int count = 0;
    for(int dr = -1; dr <= 1; dr++)
    {
        for(int dc = -1; dc <= 1; dc++)
        {
            if(dr == 0 && dc == 0) continue;
            int nr = (r + dr + rows) % rows;
            int nc = (c + dc + col) % col;
            count += grid[nr][nc];
        }
    }
    return count;
}
Key concept: The modulo operator creates wrapping edges (toroidal topology). Cells on opposite edges become neighbors, creating a seamless universe.

3. Conway's Rules

The heart of the simulation - where we apply the rules of life:

nextSim Function 
void nextSim()
{
    int next[rows][col];
    for(int r = 0; r < rows; r++)
    {
        for(int c = 0; c < col; c++)
        {
            int negi = countNeig(r, c);
            if(grid[r][c] == 1)
            {
                next[r][c] = (negi == 2 || negi == 3) ? 1 : 0;
            }
            else next[r][c] = (negi == 3) ? 1 : 0;
        }
    }
    for(int r = 0; r < rows; r++)
        for(int c = 0; c < col; c++)
            grid[r][c] = next[r][c];
}
Conway's Rules:
1. Live cell with 2-3 neighbors → survives
2. Live cell with <2 or >3 neighbors → dies
3. Dead cell with exactly 3 neighbors → becomes alive
4. Dead cell with other counts → stays dead

4. Initialization and Rendering

randomize and drawGrid Functions 
void randomize()
{
    for(int r = 0; r < rows; r++)
    {
        for(int c = 0; c < col; c++)
        {
            grid[r][c] = rand() % 2;
        }
    }
}

void drawGrid()
{
    for(int r = 0; r < rows; r++)
    {
        for(int c = 0; c < col; c++)
        {
            Color color = grid[r][c] ? GREEN : BLACK;
            DrawRectangle(c*cell_size, r*cell_size, 
                         cell_size-1, cell_size-1, color);
        }
    }
}

5. Main Program Loop

main Function 
int main()
{
    InitWindow(col*cell_size, rows*cell_size, "Game_OF_Life");
    SetTargetFPS(160);
    randomize();
    int framecount = 0;
    int speed = 5;
    
    while(!WindowShouldClose())
    {
        framecount++;
        if(framecount >= speed)
        {
            nextSim();
            framecount = 0;
        }
        BeginDrawing();
        ClearBackground(BLACK);
        drawGrid();
        EndDrawing();
    }
    CloseWindow();
    return 0;
}

The main loop creates a 1500×1000 pixel window and runs continuously. The simulation updates every 5 frames (slower than rendering) to make patterns visible.

Key Takeaways

  • Simple rules can generate complex, unpredictable behavior
  • Cellular automata demonstrate emergent properties in computation
  • Toroidal grids eliminate boundary effects
  • Synchronous updates ensure all cells change based on the same generation
  • This is foundational knowledge for game development, simulations, and AI
