Conway's Game of Life

A C++ Implementation with Raylib - Complete Code Breakdown

Understanding the Includes

Let's start with the first few lines of the code - the header includes. These tell the compiler what libraries and functionalities we need.

#include<iostream> #include<cstdlib> #include"raylib.h"

#include<iostream>

Purpose: This header provides input/output stream functionality. In our Game of Life, we might not directly use cout or cin, but it's included as a standard practice for C++ programs. It allows console output for debugging.

#include<cstdlib>

Purpose: This header provides the rand() function, which we use to randomly initialize our grid with living and dead cells. rand()%2 gives us 0 or 1 randomly.

#include"raylib.h"

Purpose: Raylib is a graphics library that handles window creation, drawing, and input. Notice the double quotes instead of angle brackets - this means it's a local library, not a system library. We use it to draw the grid, handle window events, and manage the display.

Grid Configuration & Macros

#define rows 200 #define col 300 #define cell_size 5 int grid[rows][col];

These macros define the dimensions of our cellular automaton:

  • rows = 200 - The grid has 200 rows
  • col = 300 - The grid has 300 columns
  • cell_size = 5 - Each cell is 5 pixels on the screen
  • grid[200][300] - Our 2D array where 1 = alive cell, 0 = dead cell

Note: The total window size will be 300 × 5 = 1500 pixels wide and 200 × 5 = 1000 pixels tall.

Core Functions Explained

1. countNeig(int r, int c)

This function counts the number of living neighbors around a cell at position (r, c).

int countNeig(int r, int c) { int count = 0; for(int dr = -1; dr <= 1; dr++) { for(int dc = -1; dc <= 1; dc++) { if(dr == 0 && dc == 0) continue; int nr = (r + dr + rows) % rows; int nc = (c + dc + col) % col; count += grid[nr][nc]; } } return count; }
How it works:
  • Loops through all 8 surrounding neighbors (dr and dc range from -1 to 1)
  • Skips the center cell (when dr==0 and dc==0)
  • Uses modulo arithmetic (r + dr + rows) % rows to wrap around edges (torus topology)
  • Counts living neighbors (grid value = 1)
  • Returns the count (0-8)

2. nextSim()

This is where Conway's Game of Life rules are applied!

void nextSim() { int next[rows][col]; for(int r = 0; r < rows; r++) { for(int c = 0; c < col; c++) { int negi = countNeig(r, c); if(grid[r][c] == 1) { next[r][c] = (negi == 2 || negi == 3) ? 1 : 0; } else next[r][c] = (negi == 3) ? 1 : 0; } } for(int r = 0; r < rows; r++) for(int c = 0; c < col; c++) grid[r][c] = next[r][c]; }
Conway's Rules:
Scenario Rule Result
Live cell with 2-3 neighbors Stays alive Survives
Live cell with <2 or >3 neighbors Dies (underpopulation/overpopulation) Dies
Dead cell with exactly 3 neighbors Becomes alive (reproduction) Born
Dead cell with other neighbor counts Stays dead Stays dead

3. randomize()

Initializes the grid with random alive/dead cells.

void randomize() { for(int r = 0; r < rows; r++) { for(int c = 0; c < col; c++) { grid[r][c] = rand() % 2; } } }
Explanation: Creates a random initial state by setting each cell to either 0 (dead) or 1 (alive). rand() % 2 gives a random 0 or 1.

4. drawGrid()

Renders the grid on the screen using Raylib.

void drawGrid() { for(int r = 0; r < rows; r++) { for(int c = 0; c < col; c++) { Color color = grid[r][c] ? GREEN : BLACK; DrawRectangle(c*cell_size, r*cell_size, cell_size-1, cell_size-1, color); } } }
Explanation: For each cell, draws a colored rectangle. If the cell is alive (1), it's GREEN; if dead (0), it's BLACK. Each rectangle is positioned at (c*cell_size, r*cell_size) and sized cell_size-1 for a grid effect.

The Main Loop

int main() { InitWindow(col*cell_size, rows*cell_size, "Game_OF_Life"); SetTargetFPS(160); randomize(); int framecount = 0; int speed = 5; while(!WindowShouldClose()) { framecount++; if(framecount >= speed) { nextSim(); framecount = 0; } BeginDrawing(); ClearBackground(BLACK); drawGrid(); EndDrawing(); } CloseWindow(); }

Step-by-step Breakdown:

1. InitWindow(): Creates a window with width = 300×5 = 1500 pixels and height = 200×5 = 1000 pixels

2. SetTargetFPS(160): Targets 160 frames per second for smooth rendering

3. randomize(): Sets up initial random cell states

4. Main Loop: Continues while window is open

  • framecount++ - Increments frame counter
  • if(framecount >= speed) - Updates simulation every 5 frames (slower than rendering)
  • BeginDrawing() / EndDrawing() - Raylib drawing commands
  • ClearBackground(BLACK) - Clears screen each frame
  • drawGrid() - Draws the current state

5. CloseWindow(): Cleans up and closes the window

Key Concepts to Remember

Cellular Automaton

A discrete model where each cell can be in one of a finite number of states, and the state of each cell changes over time based on its current state and its neighbors' states.

Toroidal Grid (Wrapping)

The modulo operator makes the grid wrap around at edges. Cells on the right edge have the left edge as neighbors. This creates a universe without boundaries.

Synchronous Updates

All cells update simultaneously based on the previous generation's state. This is why we use a separate next array instead of updating the original grid directly.

Visual Representation

Green cells are alive, black cells are dead. This simple visual makes it easy to spot patterns like gliders, blinkers, and beehives.
